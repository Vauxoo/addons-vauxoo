Stock Purchase Expiry

Add a contract due date to the purchase order that will control the related picking and invoice possible operations.

NOTE: This module create workarodund buttons to use the basic odoo flow without any restrictions.

Purchase Contract Due Date

Add a new field named contract due date in the Puchase Order model and show it in the purchase order form. By default this field is empty until the user manual add a contract due date.

When a picking duplication is peform the copy record will skip the value and clear the picking contract due date field.

Picking Contract Due Date

When a Purchase Order is confirm will create stock pikings, This pickings will have a contract due date pull from the Purchase Order.

Frist, Create a Purchase Order and then set a Contract Due Date. Then, Confirm the Purchase Order in order to generate the incomming pickings..

Second, Check that the Purchase Order was confirmed and go to see the Stock Pickings related by cliking the Incomming Pickings button.

Third, the picking generated via Purchase Order confirmation will have set Contract Due Date with the same date given in the Purchase Order.

Receive Products

Add a new restriction that will not let to receive products for a picking when the Contract Due Date passed. First, Create a Purchase Order and change contract due date to a past date and Confirm the order.

Second, Go inside the picking generated and click in the Receive button. An error message will be pop up saying Cannot receive a picking with a contract due date of pickings in done state.

In special ocations when want to recieve products even in the contract due date is passed a Force Receive button was added to the Stock Picking from view to force the receive products wizard as the normal action.

Create an Invoice

The invoice creation is also affected with the Contract Due Date. After the products are received the picking change to Received state and the Create Invoice/Refund button is shown in the picking form view. This button will let you to create an invoice/refund related to the given picking via the Create Draft Invoice wizard.

The mentioned wizard can also be access in the picking list view. First, select the picking you want to process and then go to the More button to display the possible actions to apply over the pickings. The Create Draft Invoice action will raise the same wizard of the picking from view described in the paragraph above.

When using the Create Draft Invoice you can regular create the invoice of the selected picking if the Contract Due Date is not passed yet. If the date is passed instead the invoice creation will fail.

A Force Create button was added in the in the Create Draft Invoices wizard. When a stock picking have contract due date then with this button will proceed no matter what and will create the invoice even if the stock picking have a contract due date.
